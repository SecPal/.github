# Support

Thank you for using SecPal! This document provides information on how to get support.

## Documentation

Before seeking support, please check our documentation:

- [README](../README.md) - Project overview and quick start
- [User Documentation](../docs/user/) - End-user guides
- [Developer Documentation](../docs/developer/) - Development guides
- [API Documentation](../docs/api/) - API reference

## Questions and Discussions

For general questions, discussions, and community support:

- **GitHub Discussions**: [SecPal Discussions](https://github.com/SecPal/SecPal/discussions)
  - Q&A - Ask questions and get help from the community
  - Ideas - Share and discuss feature ideas
  - Show and Tell - Share what you've built with SecPal

## Bug Reports

If you've found a bug, please:

1. Check if it's already reported in [Issues](https://github.com/SecPal/SecPal/issues)
2. If not, [create a new bug report](https://github.com/SecPal/SecPal/issues/new?template=bug_report.md)

## Feature Requests

To request a new feature:

1. Check if it's already requested in [Issues](https://github.com/SecPal/SecPal/issues)
2. If not, [create a feature request](https://github.com/SecPal/SecPal/issues/new?template=feature_request.md)

## Security Issues

**Do not report security vulnerabilities as public issues!**

Please report security vulnerabilities through:

- [GitHub Security Advisories](https://github.com/SecPal/SecPal/security/advisories/new)

See our [Security Policy](SECURITY.md) for more details.

## Commercial Support

For commercial support, training, or custom development:

- Contact: [support@secpal.org](mailto:support@secpal.org)
- Website: https://secpal.org/support

## Response Times

Please note that SecPal is an open source project maintained by volunteers. We'll do our best to respond quickly, but response times may vary:

- **Security Issues**: Within 24 hours
- **Bug Reports**: Within 1 week
- **Feature Requests**: Within 2 weeks
- **General Questions**: Best effort basis

## How to Ask Good Questions

To help us help you faster:

1. **Be specific**: Describe your issue in detail
2. **Provide context**: Include relevant environment information
3. **Show what you've tried**: List the solutions you've already attempted
4. **Include examples**: Provide code snippets, error messages, or screenshots
5. **Be respectful**: Remember we're all volunteers helping each other
