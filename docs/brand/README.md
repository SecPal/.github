# Brand Standards

This section contains organization-wide brand standards for SecPal.

Use these documents for shared identity guidance that applies across SecPal repositories, products, communications, and contributor-facing materials. Product-specific implementation details belong in the repository that owns the product or surface.

## Scope

Brand standards may cover:

- brand positioning and voice
- naming and terminology rules
- logo and wordmark usage
- color and typography identity guidance
- cross-repository communication conventions

Brand standards do not cover:

- product-specific component code
- repository-local asset build pipelines
- app or website implementation details
- one-off campaign copy that is not reusable guidance

## Document Layout

Current brand standards live under this directory:

- `docs/brand/README.md` - section entry point and scope boundary
- `docs/brand/brand-architecture.md` - SecPal and product-family hierarchy
- `docs/brand/naming.md` - public names, capitalization, and terminology rules
- `docs/brand/slogans.md` - approved slogan strings and punctuation rules
- `docs/brand/footer-wording.md` - approved compact footer strings
- `docs/brand/logo-usage.md` - logo usage rules and asset ownership boundaries
- `docs/brand/licensing-wording.md` - human-readable and SPDX license wording rules

Future brand standards may include:

- `docs/brand/voice-and-tone.md` - organization voice, tone, and messaging guidance
- `docs/brand/color.md` - brand color identity and usage guidance
- `docs/brand/typography.md` - brand typography identity and usage guidance

Add new brand standards here only when they describe reusable organization-wide guidance.
